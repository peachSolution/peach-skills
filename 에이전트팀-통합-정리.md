# 에이전트팀 통합 정리

`peach-skills`에서는 기존 Claude 전용 `.claude/agents` 파일을 별도 배포하지 않고, 팀 역할 정의를 팀 스킬 안으로 흡수한다.

## 통합 방향

- 기존 `team-backend`, `team-ui`, `team-fullstack`는 `peach-agent-team`으로 통합한다.
- 기존 `team-refactor`는 목적이 다르므로 `peach-agent-team-refactor`로 분리 유지한다.
- Cursor, Claude Code, Codex, OpenClaw에서 공통 사용 가능하도록 agent-neutral 문서 구조를 유지한다.

## `peach-agent-team`

신규 기능 개발용 통합 팀 스킬이다.

### 모드

- `mode=backend` : 기존 UI에 API + Store 연결
- `mode=ui` : Store 기반 UI만 구현
- `mode=fullstack` : DB 스키마 기반 전체 생성

### 포함 역할

- `backend-dev`
- `backend-qa`
- `store-dev`
- `ui-dev`
- `frontend-qa`

### 역할별 기준 스킬

| 역할 | 기준 스킬 |
| --- | --- |
| `backend-dev` | `peach-gen-backend` |
| `store-dev` | `peach-gen-store` |
| `ui-dev` | `peach-gen-ui`, `peach-gen-design` |
| `backend-qa` | 검증 전용 |
| `frontend-qa` | 검증 전용 |

### 기본 흐름

`mode=backend`
- backend-dev → backend-qa
- backend-dev → store-dev → frontend-qa

`mode=ui`
- ui-dev → frontend-qa

`mode=fullstack`
- backend-dev → backend-qa
- backend-dev → store-dev → ui-dev → frontend-qa

## `peach-agent-team-refactor`

리팩토링 전용 팀 스킬이다.

### 레이어

- `layer=backend`
- `layer=frontend`
- `layer=all`

### 포함 역할

- `refactor-backend`
- `backend-qa`
- `refactor-frontend`
- `frontend-qa`

### 역할별 기준 스킬

| 역할 | 기준 스킬 |
| --- | --- |
| `refactor-backend` | `peach-refactor-backend` |
| `refactor-frontend` | `peach-refactor-frontend` |
| `backend-qa` | 검증 전용 |
| `frontend-qa` | 검증 전용 |

### 기본 흐름

`layer=backend`
- refactor-backend → backend-qa

`layer=frontend`
- refactor-frontend → frontend-qa

`layer=all`
- refactor-backend → backend-qa
- refactor-backend → refactor-frontend → frontend-qa

## 정리 원칙

- 팀 역할 문서는 별도 `.claude/agents` 파일로 두지 않는다.
- 역할별 페르소나, 체크리스트, 완료 기준은 팀 스킬 문서에 포함한다.
- 공개 저장소에서는 설치 가능한 스킬 정본만 관리한다.
